## 2. 잘한 점

이전 리뷰 이후 정말 많은 진전이 있었어요! 페이지 3개가 새로 만들어졌고, JavaScript 컴포넌트까지 구현했어요.

### RoomCard 클래스가 인상적이에요

- `src/RoomCard.js`에서 **ES6 클래스**와 **private 필드(`#`)**를 사용한 건 정말 도전적이에요! 초보자 수준을 넘어서는 시도에요.
- `#buildElement()`로 DOM 요소를 생성하고, `#syncDOM()`으로 데이터를 화면에 반영하는 구조가 아주 깔끔해요. 데이터와 화면을 분리하는 좋은 패턴이에요.
- `set()` 메서드로 나중에 데이터를 업데이트할 수 있게 만든 것도 확장성을 고려한 좋은 설계에요.

```js
// 데이터 변경 → DOM 자동 업데이트 패턴 — 정말 잘 만들었어요!
set({ title = this.#title, ... } = {}) {
  this.#title = title;
  this.#syncDOM();
}
```

### Tailwind CSS 반응형 디자인을 잘 활용했어요

- `index.html`에서 반응형 그리드를 아주 잘 적용했어요:

```html
<ul class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 2xl:grid-cols-4 gap-5 px-[clamp(20px,4vw,80px)]">
```

- 화면 크기별로 1열 → 2열 → 3열 → 4열로 변하는 반응형 레이아웃이에요. `clamp()` 함수를 사용한 유동적 패딩도 훌륭해요!

### 숙소 카드의 인터랙션이 좋아요

- `RoomCard.js`에서 하트 아이콘에 `hover:scale-110 transition-[scale] duration-300`을 적용해서 마우스를 올리면 부드럽게 커지는 효과를 넣었어요. 사용자 경험(UX)을 고려한 디테일이에요!

### 회원가입 페이지의 label-input 연결이 정확해요

- `Signup/index.html`에서 `<label for="latteId">`와 `<input id="latteId">`를 잘 연결했어요. 이건 웹 접근성에서 정말 중요한 부분인데, 처음부터 올바르게 구현했어요!

### 숙소 상세 페이지 구조가 잘 잡혀있어요

- `rooms-detail/index.html`에서 `<section>`으로 영역을 논리적으로 잘 나누었어요 (썸네일, 기본정보, 호스트, 설명, 리뷰, 예약). Tailwind 테마 변수(`var(--color-primary-500)`)도 활용하고 있어요.

## 3. 개선하면 좋을 점

### 3-1. 회원가입 페이지에 `<form>` 태그가 없어요 (중요도: 높음)

**현재 상태:**

`Signup/index.html`에서 입력 필드들이 `<form>` 없이 바로 `<body>` 안에 있어요:

```html
<body>
  <p>에어비앤비에 오신 것을 환영합니다!</p>
  <div id="id">
    <label for="latteId">아이디</label>
    <input type="text" id="latteId" ... />
  </div>
  ...
  <button type="submit" id="lattebtn">회원가입</button>
</body>
```

**왜 개선하면 좋을까?**

`<button type="submit">`은 `<form>` 안에 있어야 작동해요. `<form>` 없이는 버튼을 눌러도 아무 일도 일어나지 않아요. 또한 `<form>`이 있어야 Enter 키로 전송하기, 브라우저 자동완성 같은 기본 기능이 동작해요.

**추천:** 입력 필드들을 `<form>` 태그로 감싸세요:

```html
<body>
  <p>에어비앤비에 오신 것을 환영합니다!</p>
  <form>
    <div id="id">
      <label for="latteId">아이디</label>
      <input type="text" id="latteId" ... />
    </div>
    ...
    <button type="submit" id="lattebtn">회원가입</button>
  </form>
</body>
```

### 3-2. 비밀번호 입력란이 `type="text"`로 되어 있어요 (중요도: 높음)

**현재 상태:**

```html
<!-- Signup/index.html 75번째 줄 -->
<input type="text" id="lattePw" placeholder="비밀번호를 입력해주세요." />
```

**왜 개선하면 좋을까?**

비밀번호를 `type="text"`로 하면 입력 시 비밀번호가 그대로 화면에 보여요. `type="password"`로 하면 `●●●●●`로 가려져서 보안이 훨씬 좋아져요. 전화번호도 `type="tel"`을 쓰면 모바일에서 숫자 키패드가 자동으로 표시돼요.

**추천:**

```html
<!-- 비밀번호: type="password"로 변경 -->
<input type="password" id="lattePw" placeholder="비밀번호를 입력해주세요." />

<!-- 전화번호: type="tel"로 변경 -->
<input type="tel" id="lattePn" placeholder="예) 010-1234-5678" />
```

### 3-3. 회원가입 페이지가 Tailwind CSS를 사용하지 않아요 (중요도: 높음)

**현재 상태:**

`Signup/index.html`은 `<style>` 태그로 인라인 CSS를 작성하고 있고, Tailwind CSS를 전혀 사용하지 않아요:

```html
<style>
  body {
    display: flex;
    flex-flow: nowrap column;
    ...
  }
  #lattebtn {
    background: #ff8194;
    ...
  }
</style>
```

반면 `index.html`과 `rooms-detail/index.html`은 Tailwind CSS를 사용하고 있어요.

**왜 개선하면 좋을까?**

팀 프로젝트에서 한 페이지는 Tailwind, 다른 페이지는 인라인 CSS를 쓰면:
- 스타일이 일관되지 않아요 (같은 버튼인데 다르게 보일 수 있어요)
- 나중에 유지보수가 힘들어져요 (두 가지 방식을 다 알아야 하니까)
- 테마 색상(`var(--color-primary-500)`)을 활용할 수 없어요 (현재 `#ff8194`를 하드코딩했어요)

**추천:** `Signup/index.html`에도 Tailwind CSS를 도입하세요:

1. `<head>`에 Tailwind CSS를 로드하는 스크립트 추가
2. `<style>` 태그를 제거하고 Tailwind 유틸리티 클래스로 변환

```html
<head>
  ...
  <script type="module" src="/src/main.js"></script>
</head>
```

### 3-4. RoomCard.js에 오타가 있어서 Tailwind 클래스가 작동 안 해요 (중요도: 높음)

**현재 상태:**

```js
// src/RoomCard.js 50번째 줄
<p class="accommodationTitle text-lg col-satrt-1"></p>
```

`col-satrt-1`이라고 되어 있는데, 올바른 Tailwind 클래스는 `col-start-1`이에요 (`satrt` → `start`).

**왜 개선하면 좋을까?**

Tailwind CSS는 클래스 이름을 정확히 써야 해요. 한 글자라도 다르면 스타일이 전혀 적용되지 않아요. `col-satrt-1`은 존재하지 않는 클래스라서 그리드 배치가 의도대로 동작하지 않아요.

**추천:**

```js
// 수정 전
<p class="accommodationTitle text-lg col-satrt-1"></p>

// 수정 후
<p class="accommodationTitle text-lg col-start-1"></p>
```

**현재 상태:**

```js
// src/RoomCard.js 8번째 줄
```


### 3-7. rooms-detail 페이지에 CSS 정의 없는 클래스가 있어요 (중요도: 중간)

**현재 상태:**

`rooms-detail/index.html`에 이런 클래스들이 사용되고 있어요:

```html
<section class="rooms-detail-thumb">
<section class="rooms-detail-info">
<p class="rooms-detail-info-address">
<div class="host-profile">
<div class="review-card">
```

그런데 이 클래스들에 대한 CSS가 어디에도 정의되어 있지 않아요. Tailwind CSS의 유틸리티 클래스도 아니고, 별도의 CSS 파일도 없어요.

**왜 개선하면 좋을까?**

CSS 정의가 없는 클래스는 아무런 스타일 효과가 없어요. 이름만 있고 실제로는 아무 것도 안 하는 거에요. 나중에 스타일을 추가하려고 남겨둔 거라면 괜찮지만, Tailwind CSS를 사용하고 있으니 Tailwind 유틸리티 클래스로 스타일링하는 것이 일관성 있는 방법이에요.

**추천:** 두 가지 방법 중 하나를 선택하세요:

1. **Tailwind 유틸리티 클래스로 직접 스타일링** (추천):
```html
<section class="w-full">
  <img class="w-full h-full object-cover" src="..." alt="썸네일 이미지" />
</section>
```

2. **Tailwind의 `@apply`로 커스텀 클래스 정의** (`style.css`에 추가):
```css
.rooms-detail-thumb {
  @apply w-full;
}
.rooms-detail-thumb-img {
  @apply w-full h-full object-cover;
}
```

### 3-8. index.html에 시맨틱 태그가 빠져 있어요 (중요도: 중간)

**현재 상태:**

```html

<body>
  <h1 class="...">Rooms | 특별한 추억을 남길 수 있는 공간을 찾아보세요.</h1>
```

`<header>`, `<main>`, `<nav>`, `<footer>` 같은 시맨틱 태그가 없어요.

**왜 개선하면 좋을까?**

시맨틱 태그는 웹 페이지의 "의미 있는 뼈대"에요. 없어도 화면에는 보이지만:
- 스크린 리더(시각장애인용 도구)가 페이지 구조를 이해할 수 없어요
- 검색엔진이 콘텐츠를 제대로 파악하기 어려워요
- 나중에 헤더나 푸터를 추가할 때 구조가 복잡해져요

**추천:**

```html
<body>
  <header>
    <nav><!-- 네비게이션 바 (로고, 검색, 로그인 등) --></nav>
  </header>
  <main>
    <h1 class="...">Rooms | 특별한 추억을 남길 수 있는 공간을 찾아보세요.</h1>
    <ul id="roomList" class="..."></ul>
  </main>
  <footer><!-- 푸터 --></footer>
  <script type="module" src="./src/main.js"></script>
  <script type="module" src="./src/landing.js"></script>
</body>
```

개발 중에 `console.log`로 값을 확인하는 건 좋은 습관이에요! 하지만 작업이 끝나면 지워주는 게 좋아요. 브라우저 콘솔에 불필요한 로그가 쌓이면 나중에 진짜 중요한 로그를 찾기 어려워져요.

**추천:** `console.log(i);` 줄을 삭제하세요.

### 3-10. RoomCard에서 저장하지만 사용하지 않는 데이터가 있어요 (중요도: 낮음)

**현재 상태:**

`RoomCard.js`에서 `location`, `reviewCount`, `hostName`을 저장하지만 화면에 표시하지 않아요:

```js
this.#location = location;      // 저장만 하고 표시 안 함
this.#reviewCount = reviewCount; // 저장만 하고 표시 안 함
this.#hostName = hostName;       // 저장만 하고 표시 안 함
```

**왜 개선하면 좋을까?**

나중에 사용할 예정이라면 괜찮아요! 하지만 ESLint 설정에서 `no-unused-vars: 'warn'`을 켜두었듯이, 사용하지 않는 데이터는 혼란을 줄 수 있어요.

**추천:** 카드에 위치와 호스트 이름도 표시하면 에어비앤비처럼 더 정보가 풍부한 카드가 돼요. `#buildElement()` 안의 HTML 템플릿에 추가해보세요!

### 3-11. 가격에 천 단위 콤마가 없어요 (중요도: 낮음)

**현재 상태:**

```js
// src/RoomCard.js 87번째 줄
this.#priceNode.textContent = `₩ ${this.#audultPricePerNight} / 1박`;
// 결과: "₩ 100000 / 1박"
```

**왜 개선하면 좋을까?**

"₩ 100000"보다 "₩ 100,000"이 훨씬 읽기 쉬워요. JavaScript의 `toLocaleString()`을 쓰면 자동으로 콤마를 넣어줘요.

**추천:**

```js
this.#priceNode.textContent = `₩ ${this.#audultPricePerNight.toLocaleString()} / 1박`;
// 결과: "₩ 100,000 / 1박"
```
