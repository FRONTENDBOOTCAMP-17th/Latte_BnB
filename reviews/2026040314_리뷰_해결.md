## 3-1. 회원가입 전화번호 유효성 검사에 버그가 있어요 (중요도: 높음)

### 문제

`signup/index.js` 83번째 줄에서 `&&`(AND) 연산자를 사용하고 있어서, 비어있지 않은 잘못된 전화번호(예: `"abc"`)가 유효성 검사를 통과해요.

### 해결 방법

**Step 1.** `signup/index.js` 83번째 줄의 `&&`를 `||`로 변경하세요.

```js
// 수정 전 (signup/index.js 83번째 줄)
if (!signupData.phone && !regPn.test(signupData.phone)) {

// 수정 후
if (!signupData.phone || !regPn.test(signupData.phone)) {
```

`||`(OR)을 쓰면:
- `""` (빈 값): `!""`는 `true` → 에러 표시 (정상)
- `"abc"` (잘못된 값): `!"abc"`는 `false`, `!regPn.test("abc")`는 `true` → 에러 표시 (정상!)
- `"01012345678"` (정상): 둘 다 `false` → 통과 (정상)

> **쉽게 말하면:** "비어있**거나** 형식이 안 맞으면" 에러를 보여주는 게 맞아요. `&&`는 "비어있**고 동시에** 형식도 안 맞으면"이라서 조건이 너무 까다로워져요!

## 3-2. 로그인 성공 alert가 토큰 저장보다 먼저 실행돼요 (중요도: 높음)

### 문제

`login/index.js` 71~81번째 줄에서 `alert('로그인 되었습니다.')`가 토큰 저장과 페이지 이동보다 먼저 실행돼요.

### 해결 방법

**Step 1.** `login/index.js`의 try 블록 내용을 재배치하세요.

```js
// 수정 전 (login/index.js 59~85번째 줄)
try {
  const res = await fetch(
    'https://api.fullstackfamily.com/api/lattebnb/v1/auth/login',
    {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify(loginData),
    },
  );

  if (!res.ok) {
    throw new Error('HTTP 오류: ' + res.status);
  } else {
    alert(`로그인 되었습니다.`);
  }

  const data = await res.json();
  const personalData = data.data;
  localStorage.setItem('accessToken', personalData.accessToken);

  location.href = `../`;
} catch (e) {
  result3.textContent = '아이디 또는 비밀번호를 다시 입력해주세요.';
  result3.style.color = 'red';
}

// 수정 후
try {
  const res = await fetch(
    'https://api.fullstackfamily.com/api/lattebnb/v1/auth/login',
    {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify(loginData),
    },
  );

  if (!res.ok) {
    throw new Error('HTTP 오류: ' + res.status);
  }

  const data = await res.json();
  const personalData = data.data;
  localStorage.setItem('accessToken', personalData.accessToken);

  alert(`로그인 되었습니다.`);  // 토큰 저장 완료 후!
  location.href = `../`;
} catch (e) {
  result3.textContent = '아이디 또는 비밀번호를 다시 입력해주세요.';
}
```

> **쉽게 말하면:** 은행에서 "송금 완료!" 메시지를 보여주려면, 실제로 돈이 이체된 다음에 보여줘야 해요. 이체하기 전에 "완료!"라고 하면 안 되잖아요!

## 3-3. toast.js에서 `buildMessage` 함수가 자기 자신을 호출해요 (중요도: 높음)

### 문제

`src/components/toast.js` 68번째 줄에서 `buildMessage()`가 자기 자신을 호출하여 무한 재귀가 발생해요. 74번째 줄의 `buildDetail()`은 존재하지 않는 함수에요.

### 해결 방법

## 3-5. login/signup 페이지에서 API URL을 하드코딩하고 있어요 (중요도: 중간)

### 문제

`login/index.js` 61번째 줄과 `signup/index.js` 91번째 줄에서 API URL을 직접 적고 있어요. `src/constants.js`를 활용하지 않고 있어요.

### 해결 방법

**Step 1.** `login/index.js` 파일 맨 위에 import를 추가하세요. `<script>` 태그에 `type="module"`이 있어야 import가 동작해요.

```js
// 수정 전 (login/index.js 1번째 줄)
const loginbtn = document.getElementById('loginbtn');

// 수정 후 — 맨 위에 import 추가
import constants from '../src/constants.js';

const API_BASE = constants.API_BASE_URL;
const loginbtn = document.getElementById('loginbtn');
```

**Step 2.** fetch URL을 상수로 변경하세요.

```js
// 수정 전 (login/index.js 61번째 줄)
const res = await fetch(
  'https://api.fullstackfamily.com/api/lattebnb/v1/auth/login',
  ...
);

// 수정 후
const res = await fetch(`${API_BASE}/auth/login`, ...);
```

**Step 3.** `login/index.html`의 script 태그에 `type="module"`이 이미 있는지 확인하세요. 있으면 그대로, 없으면 추가하세요:

```html
<script type="module" src="./index.js"></script>
```

**Step 4.** `signup/index.js`에도 동일하게 적용하세요.

```js
// signup/index.js 맨 위에 추가
import constants from '../src/constants.js';

const API_BASE = constants.API_BASE_URL;
```

```js
// 수정 전 (signup/index.js 91번째 줄)
const res = await fetch(
  'https://api.fullstackfamily.com/api/lattebnb/v1/auth/signup',
  ...
);

// 수정 후
const res = await fetch(`${API_BASE}/auth/signup`, ...);
```

> **쉽게 말하면:** 반 친구 전화번호를 매번 외워서 치는 것보다, 연락처에 저장해두고 이름으로 검색하는 게 편하잖아요. `constants.js`가 바로 그 연락처에요!

## 3-8. accommodations-detail.js에서 API 데이터를 innerHTML에 직접 넣고 있어요 (중요도: 중간)

### 문제

`accommodations-detail/accommodations-detail.js` 34~39번째 줄에서 API 응답 데이터를 `innerHTML`에 직접 삽입하고 있어요.

### 해결 방법

**Step 1.** innerHTML로 구조만 만들고, 데이터는 `textContent`와 속성 설정으로 넣으세요.

```js
// 수정 전 (accommodations-detail/accommodations-detail.js 32~39번째 줄)
data.images.forEach((img) => {
  const figure = document.createElement('figure');
  figure.className = 'min-w-50 snap-start m-0';
  figure.innerHTML =
    `<img src="${img.url}" alt="${img.title || '숙소 이미지'}" class="w-full aspect-[4/3] object-cover rounded-lg" />` +
    `<figcaption class="flex flex-col mt-1.5">` +
    `<span class="text-sm font-semibold text-shark-800">${img.title || ''}</span>` +
    `<span class="text-[13px] text-shark-500">${img.description || ''}</span>` +
    `</figcaption>`;
  gallery.appendChild(figure);
});

// 수정 후
data.images.forEach((img) => {
  const figure = document.createElement('figure');
  figure.className = 'min-w-50 snap-start m-0';
  figure.innerHTML =
    `<img class="w-full aspect-[4/3] object-cover rounded-lg" />` +
    `<figcaption class="flex flex-col mt-1.5">` +
    `<span class="title text-sm font-semibold text-shark-800"></span>` +
    `<span class="desc text-[13px] text-shark-500"></span>` +

  // 데이터는 textContent/속성으로 안전하게 삽입
  const imgEl = figure.querySelector('img');

  figure.querySelector('.title').textContent = img.title || '';
```

아래의 placeList에도 동일한 패턴을 적용하세요 (42~50번째 줄).

> **쉽게 말하면:** 편지 봉투(HTML 구조)는 미리 만들어두고, 내용물(데이터)은 안전하게 넣는 거에요. 봉투 안에 폭죽(스크립트)이 들어가도 터지지 않게!
