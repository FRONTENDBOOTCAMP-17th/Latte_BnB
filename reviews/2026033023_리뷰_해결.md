## 3-1. 회원가입 페이지에 `<form>` 태그가 없어요 (중요도: 높음)

### 문제

`Signup/index.html` 67~86번째 줄에서 입력 필드와 제출 버튼이 `<form>` 밖에 있어서 `<button type="submit">`이 작동하지 않아요.

### 해결 방법

**Step 1.** `Signup/index.html`에서 입력 필드들을 `<form>` 태그로 감싸세요.

```html
<!-- 수정 전 (Signup/index.html 67~86번째 줄) -->
<body>
  <p>에어비앤비에 오신 것을 환영합니다!</p>
  <div id="id">
    <label for="latteId">아이디</label>
    <input type="text" id="latteId" placeholder="아이디를 입력해주세요." />
  </div>
  <div id="pwd">
    <label for="lattePw">비밀번호</label>
    <input type="text" id="lattePw" placeholder="비밀번호를 입력해주세요." />
  </div>
  <div id="name">
    <label for="latteNm">이름</label>
    <input type="text" id="latteNm" placeholder="이름을 입력해주세요." />
  </div>
  <div id="phone">
    <label for="lattePn">전화번호</label>
    <input type="text" id="lattePn" placeholder="예) 010-1234-5678" />
  </div>
  <button type="submit" id="lattebtn">회원가입</button>
</body>

<!-- 수정 후 -->
<body>
  <p>에어비앤비에 오신 것을 환영합니다!</p>
  <form>
    <div id="id">
      <label for="latteId">아이디</label>
      <input type="text" id="latteId" placeholder="아이디를 입력해주세요." />
    </div>
    <div id="pwd">
      <label for="lattePw">비밀번호</label>
      <input type="password" id="lattePw" placeholder="비밀번호를 입력해주세요." />
    </div>
    <div id="name">
      <label for="latteNm">이름</label>
      <input type="text" id="latteNm" placeholder="이름을 입력해주세요." />
    </div>
    <div id="phone">
      <label for="lattePn">전화번호</label>
      <input type="tel" id="lattePn" placeholder="예) 010-1234-5678" />
    </div>
    <button type="submit" id="lattebtn">회원가입</button>
  </form>
</body>
```

**Step 2.** `<form>`에 맞게 `<style>`도 업데이트하세요. 기존 `body`에 적용했던 flex 관련 스타일을 `form`으로 옮기면 돼요:

```css
/* 수정 전 */
body {
  display: flex;
  flex-flow: nowrap column;
  justify-content: center;
  gap: 3rem;
  padding-block: 2rem;
  margin: 0;
}

/* 수정 후 — body와 form 모두에 스타일 적용 */
body {
  margin: 0;
  padding-block: 2rem;

  @media (width >= 768px) {
    inline-size: 450px;
    justify-self: center;
  }
  @media (width >= 1920px) {
    inline-size: 600px;
    justify-self: center;
  }
}

form {
  display: flex;
  flex-flow: nowrap column;
  justify-content: center;
  gap: 3rem;
}
```

> **쉽게 말하면:** `<form>`은 서류를 담는 봉투 같아요. 서류(입력 필드)들을 봉투에 넣지 않으면 제출(submit) 버튼을 눌러도 서류가 전달되지 않아요!

## 3-2. 비밀번호 입력란이 `type="text"`로 되어 있어요 (중요도: 높음)

### 문제

`Signup/index.html` 75번째 줄에서 비밀번호 입력란이 `type="text"`로 되어 있어 비밀번호가 화면에 그대로 보여요. 83번째 줄의 전화번호도 `type="text"`에요.

### 해결 방법

**Step 1.** 비밀번호 input의 type을 변경하세요.

```html
<!-- 수정 전 (Signup/index.html 75번째 줄) -->
<input type="text" id="lattePw" placeholder="비밀번호를 입력해주세요." />

<!-- 수정 후 -->
<input type="password" id="lattePw" placeholder="비밀번호를 입력해주세요." />
```

**Step 2.** 전화번호 input의 type도 변경하세요.

```html
<!-- 수정 전 (Signup/index.html 83번째 줄) -->
<input type="text" id="lattePn" placeholder="예) 010-1234-5678" />

<!-- 수정 후 -->
<input type="tel" id="lattePn" placeholder="예) 010-1234-5678" />
```

`type="password"` → 입력할 때 `●●●●●`로 가려져요
`type="tel"` → 모바일에서 숫자 키패드가 자동으로 뜨고, 전화번호 자동완성도 돼요

> **쉽게 말하면:** ATM에서 비밀번호 칠 때 화면에 숫자가 보이면 무섭잖아요. `type="password"`는 비밀번호를 가려주는 가림막이에요!

## 3-3. 회원가입 페이지가 Tailwind CSS를 사용하지 않아요 (중요도: 높음)

### 문제

`Signup/index.html`은 `<style>` 태그로 CSS를 작성하고 있고, 프로젝트의 Tailwind CSS 테마 색상도 사용하지 않아요. 버튼 색상 `#ff8194`가 하드코딩되어 있어요 (57번째 줄).

### 해결 방법

**Step 1.** `<head>`에 Tailwind CSS를 사용할 수 있도록 `main.js`를 연결하세요.

```html
<!-- 수정 전 (Signup/index.html 3~6번째 줄) -->
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>회원가입 페이지</title>
  <style>
    ...65줄의 CSS...
  </style>
</head>

<!-- 수정 후 -->
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>회원가입 - 라떼비앤비</title>
  <script type="module" src="/src/main.js"></script>
</head>
```

**Step 2.** `<style>` 태그를 삭제하고, HTML 요소에 Tailwind 유틸리티 클래스를 적용하세요.

```html
<!-- 수정 후 body 예시 -->
<body class="flex flex-col justify-center gap-12 py-8 mx-auto md:w-[450px] 2xl:w-[600px]">
  <p class="flex justify-center text-2xl font-bold">
    에어비앤비에 오신 것을 환영합니다!
  </p>
  <form class="flex flex-col gap-8">
    <div class="flex flex-col gap-2">
      <label for="latteId" class="ml-4 text-shark-500">아이디</label>
      <input type="text" id="latteId"
        class="py-3 pl-3 mx-5 rounded-lg border border-shark-300"
        placeholder="아이디를 입력해주세요." />
    </div>
    <!-- 나머지 필드도 동일한 패턴으로 -->
    <button type="submit"
      class="bg-primary-400 rounded-lg self-center w-40 p-4">
      회원가입
    </button>
  </form>
</body>
```

`#ff8194`를 직접 쓰는 대신 `bg-primary-400`처럼 테마 색상을 사용하면, 나중에 색상을 바꿀 때 `style.css` 한 곳만 수정하면 모든 페이지에 반영돼요.

> **쉽게 말하면:** 팀 유니폼을 맞췄는데 한 명만 다른 옷을 입은 것과 같아요. 모든 페이지가 같은 방식(Tailwind)으로 스타일링해야 팀 프로젝트답게 통일감이 생겨요!

## 3-5. RoomCard.js에 속성 이름 오타가 있어요 (중요도: 중간)

### 문제

`src/RoomCard.js`와 `src/landing.js`에서 두 가지 속성 이름에 오타가 있어요:
- `audultPricePerNight` (RoomCard.js 8, 34, 87, 97, 106, 112번째 줄 / landing.js 17번째 줄)
- `tumbnailUrl` (RoomCard.js 28, 37, 105, 115번째 줄 / landing.js 20번째 줄)

### 해결 방법

**Step 1.** `src/RoomCard.js`에서 찾기-바꾸기(Ctrl+H 또는 Cmd+H)로 일괄 수정하세요:
- `audultPricePerNight` → `adultPricePerNight`
- `tumbnailUrl` → `thumbnailUrl`

```js
// 수정 전 (src/RoomCard.js 8번째 줄)
#audultPricePerNight;

// 수정 후
#adultPricePerNight;
```

```js
// 수정 전 (src/RoomCard.js 28번째 줄, constructor 파라미터)
tumbnailUrl = '',

// 수정 후
thumbnailUrl = '',
```

**Step 2.** `src/landing.js`에서도 동일하게 수정하세요:

```js
// 수정 전 (src/landing.js 17번째 줄)
audultPricePerNight: 100000,

// 수정 후
adultPricePerNight: 100000,
```

```js
// 수정 전 (src/landing.js 20번째 줄)
tumbnailUrl: '/src/assets/room_thumbnail.webp',

// 수정 후
thumbnailUrl: '/src/assets/room_thumbnail.webp',
```

**주의!** 두 파일 모두 수정해야 해요. 한쪽만 바꾸면 이름이 안 맞아서 에러가 나요.

> **쉽게 말하면:** 팀원끼리 "홍길동"이라는 이름을 부를 때, 한 명은 "홍길동" 다른 한 명은 "홍길돈"이라고 부르면 대화가 안 통하잖아요. 코드에서도 이름이 정확히 같아야 소통이 돼요!

## 3-10. RoomCard에서 저장하지만 사용하지 않는 데이터가 있어요 (중요도: 낮음)

```js
  <p class="accommodationTitle text-lg col-start-1"></p>
  <p class="accommodationLocation text-sm text-shark-500 col-start-1"></p>
  ...
</div>
```

**Step 2.** `#syncDOM()`에 위치 데이터 연결을 추가하세요.

```js
// #syncDOM() 안에 추가
this.#locationNode.textContent = this.#location;
```

> **쉽게 말하면:** 재료는 사왔는데 요리에 안 넣은 것과 같아요. 아직 안 넣을 거면 괜찮지만, 넣으면 더 맛있는 요리(카드)가 완성돼요!

## 3-11. 가격에 천 단위 콤마가 없어요 (중요도: 낮음)

### 문제

`src/RoomCard.js` 87번째 줄에서 가격이 "₩ 100000 / 1박"으로 콤마 없이 표시돼요.

### 해결 방법

**Step 1.** `#syncDOM()` 안의 가격 표시 부분을 수정하세요.

```js
// 수정 전 (src/RoomCard.js 87번째 줄)
this.#priceNode.textContent = `₩ ${this.#audultPricePerNight} / 1박`;

// 수정 후
this.#priceNode.textContent = `₩ ${this.#adultPricePerNight.toLocaleString()} / 1박`;
// 결과: "₩ 100,000 / 1박"
```

`toLocaleString()`은 JavaScript의 내장 메서드로, 숫자에 자동으로 천 단위 콤마를 넣어줘요.

> **쉽게 말하면:** 100000원이라고 써있으면 "10만원? 100만원?" 헷갈리잖아요. 100,000원이라고 쓰면 바로 "아, 10만 원!" 하고 알 수 있어요.
