## 3-1. primary와 secondary 색상 팔레트가 거의 동일해요 (중요도: 높음)

### 문제

`src/style.css`의 3~26번째 줄에서 `--color-primary-*`와 `--color-secondary-*`가 거의 같은 값으로 정의되어 있어요. 11개 단계 중 9개가 완전히 동일하고, 나머지 2개(400, 700)도 미세한 차이만 있어요.

### 해결 방법

**Step 1.** 팀원들과 Figma 디자인을 확인하며, secondary 색상의 역할을 정해주세요.

일반적으로 이렇게 나눌 수 있어요:
- **primary**: 브랜드 주 색상 (CTA 버튼, 중요한 강조 등)
- **secondary**: 보조 색상 (보조 버튼, 배경 강조, 태그 등)
- **tertiary**: 추가 악센트 색상

**Step 2.** secondary 색상을 primary와 구별되는 색으로 변경하세요.

```css
/* 수정 전 (src/style.css 16~26번째 줄) */
--color-secondary-50: #fff1f2;
--color-secondary-100: #ffe3e5;
--color-secondary-200: #ffccd3;
--color-secondary-300: #ffa1ad;
--color-secondary-400: #ff6d83;
--color-secondary-500: #f93a5c;
--color-secondary-600: #e71746;
--color-secondary-700: #c30d3a;
--color-secondary-800: #a30e38;
--color-secondary-900: #8b1037;
--color-secondary-950: #4e0319;

/* 수정 후 (예시: 따뜻한 회색 계열로 변경) */
/* ⚠️ 아래는 예시이에요. Figma 디자인에 맞는 색상을 사용하세요! */
--color-secondary-50: #fafafa;
--color-secondary-100: #f5f5f5;
--color-secondary-200: #e5e5e5;
--color-secondary-300: #d4d4d4;
--color-secondary-400: #a3a3a3;
--color-secondary-500: #737373;
--color-secondary-600: #525252;
--color-secondary-700: #404040;
--color-secondary-800: #262626;
--color-secondary-900: #171717;
--color-secondary-950: #0a0a0a;
```

**Step 3.** 만약 의도적으로 같은 계열이 맞다면, secondary를 삭제하고 primary 하나로 통일하는 것도 방법이에요. 불필요한 색상이 있으면 나중에 "이 버튼은 primary를 써야 하나 secondary를 써야 하나?" 하고 헷갈릴 수 있거든요.

> **쉽게 말하면:** 물감 팔레트에 빨강 물감이 두 통 있으면 어떤 걸 써야 할지 헷갈리잖아요. 역할이 다른 색은 확실히 다른 색으로 구분하는 게 좋아요!

## 3-2. README의 파일 링크에 `.md`가 중복되어 있어요 (중요도: 높음)

### 문제

`README.md` 49~53번째 줄에서 4개 링크의 파일 확장자가 `.md.md`로 되어 있어요. 실제 파일은 `.md`로 끝나기 때문에 링크가 깨져요.

### 해결 방법

**Step 1.** `README.md`를 열어서 49~53번째 줄의 링크를 수정하세요.

```html
<!-- 수정 전 (README.md 49번째 줄) -->
<li><a href="./docs/prototype-design-draft/accommodation_detail.md.md">숙소 상세 페이지</a></li>

<!-- 수정 후 -->
<li><a href="./docs/prototype-design-draft/accommodation_detail.md">숙소 상세 페이지</a></li>
```

```html
<!-- 수정 전 (README.md 50번째 줄) -->
<li><a href="./docs/prototype-design-draft/reservation_request.md.md">예약 신청 페이지</a></li>

<!-- 수정 후 -->
<li><a href="./docs/prototype-design-draft/reservation_request.md">예약 신청 페이지</a></li>
```

```html
<!-- 수정 전 (README.md 51번째 줄) -->
<li><a href="./docs/prototype-design-draft/reservation_detail.md.md">예약 상세 페이지</a></li>

<!-- 수정 후 -->
<li><a href="./docs/prototype-design-draft/reservation_detail.md">예약 상세 페이지</a></li>
```

```html
<!-- 수정 전 (README.md 52번째 줄) -->
<li><a href="./docs/prototype-design-draft/my_reservation.md.md">내 예약 목록 페이지</a></li>

<!-- 수정 후 -->
<li><a href="./docs/prototype-design-draft/my_reservation.md">내 예약 목록 페이지</a></li>
```

> **쉽게 말하면:** 주소를 적을 때 "서울시 강남구구"라고 쓰면 우편이 안 가잖아요. 파일 이름 끝에 `.md`가 한 번만 있어야 올바른 주소에요!

## 3-3. README의 프로젝트 구조가 실제 구조와 달라요 (중요도: 중간)

### 문제

`README.md` 61~82번째 줄의 프로젝트 구조가 `css/`, `js/`, `assets/` 폴더 기반으로 되어 있지만, 실제로는 `src/` 폴더에 `main.js`와 `style.css`만 있어요.

### 해결 방법

**Step 1.** 팀원들과 실제 사용할 폴더 구조를 확정하세요.

Vite 기반 프로젝트에서 추천하는 구조 예시:

```
latte-bnb/
├── index.html              ← 메인 페이지
├── detail.html             ← 숙소 상세 페이지
├── wishlist.html           ← 위시리스트 페이지
├── login.html              ← 로그인 페이지
├── src/
│   ├── main.js             ← 공통 JavaScript 진입점
│   ├── style.css           ← Tailwind CSS + 테마
│   ├── js/
│   │   ├── detail.js       ← 숙소 상세 페이지 JS
│   │   └── wishlist.js     ← 위시리스트 페이지 JS
│   └── assets/
│       ├── images/         ← 이미지 파일
│       └── icons/          ← 아이콘 파일
└── docs/                   ← 문서
```

**Step 2.** 구조가 확정되면 `README.md` 61~82번째 줄을 실제 구조에 맞게 업데이트하세요.

> **쉽게 말하면:** 건물 안내판에 "3층 카페"라고 적혀있는데 실제로 3층에 가면 카페가 없으면 당황하잖아요. README는 프로젝트의 안내판이니까 실제와 똑같이 맞춰야 해요!

## 3-4. Vite 멀티 페이지 설정이 필요해요 (중요도: 중간)

### 문제

`vite.config.js`에 멀티 페이지 빌드 설정이 없어서, `npm run build`를 실행하면 `index.html` 한 페이지만 빌드돼요.

### 해결 방법

**Step 1.** 두 번째 HTML 페이지를 추가할 때, `vite.config.js`를 아래와 같이 수정하세요.

```js
// 수정 전 (vite.config.js)
import { defineConfig } from 'vite';
import tailwindcss from '@tailwindcss/vite';

export default defineConfig({
  plugins: [tailwindcss()],
});

// 수정 후
import { resolve } from 'path';
import { defineConfig } from 'vite';
import tailwindcss from '@tailwindcss/vite';

export default defineConfig({
  plugins: [tailwindcss()],
  build: {
    rollupOptions: {
      input: {
        main: resolve(__dirname, 'index.html'),
        detail: resolve(__dirname, 'detail.html'),
        wishlist: resolve(__dirname, 'wishlist.html'),
        login: resolve(__dirname, 'login.html'),
      },
    },
  },
});
```

**Step 2.** 새로운 HTML 파일(예: `detail.html`)을 프로젝트 루트에 만들 때마다 `input` 객체에 추가하세요.

> **쉽게 말하면:** Vite한테 "나 페이지 여러 개 만들 거야, 이것들 다 빌드해줘"라고 목록을 알려주는 거에요. 목록에 없는 페이지는 빌드할 때 빠져버려요!

## 3-5. `<title>` 태그를 더 설명적으로 작성하면 좋아요 (중요도: 낮음)

### 문제

`index.html` 6번째 줄의 `<title>`이 `latte-bnb`로만 되어 있어요.

### 해결 방법

**Step 1.** `index.html`의 6번째 줄을 수정하세요.

```html
<!-- 수정 전 (index.html 6번째 줄) -->
<title>latte-bnb</title>

<!-- 수정 후 -->
<title>라떼비앤비 - 특별한 숙소를 찾아보세요</title>
```

**Step 2.** 각 페이지마다 해당 페이지에 맞는 제목을 넣으세요.

```html
<!-- detail.html -->
<title>숙소 상세 - 라떼비앤비</title>

<!-- wishlist.html -->
<title>위시리스트 - 라떼비앤비</title>

<!-- login.html -->
<title>로그인 - 라떼비앤비</title>
```

> **쉽게 말하면:** 브라우저 탭은 책의 제목 같은 거에요. "책"이라고만 적혀있으면 무슨 책인지 모르잖아요. "해리포터 - 마법사의 돌"처럼 구체적으로 적어야 한눈에 알 수 있어요!

## 3-6. `index.html`의 `<body>`에 기본 콘텐츠가 없어요 (중요도: 낮음)

### 문제

`index.html` 8~10번째 줄의 `<body>`에 스크립트 태그만 있고 HTML 콘텐츠가 없어요.

### 해결 방법

**Step 1.** 메인 페이지의 기본 뼈대를 시맨틱 태그(의미를 가진 태그)로 작성하세요.

```html
<!-- 수정 전 (index.html 8~10번째 줄) -->
<body>
  <script type="module" src="/src/main.js"></script>
</body>

<!-- 수정 후 -->
<body>
  <header>
    <nav>
      <!-- 로고, 검색바, 로그인 버튼 등 -->
    </nav>
  </header>

  <main>
    <section>
      <!-- 카테고리 필터 -->
    </section>
    <section>
      <!-- 숙소 카드 목록 -->
    </section>
  </main>

  <footer>
    <!-- 하단 정보 -->
  </footer>

  <script type="module" src="/src/main.js"></script>
</body>
```

시맨틱 태그 간단 설명:
- `<header>`: 페이지 상단 영역 (로고, 네비게이션)
- `<nav>`: 네비게이션 메뉴
- `<main>`: 페이지의 주요 콘텐츠 (한 페이지에 하나만!)
- `<section>`: 콘텐츠의 구분된 영역
- `<footer>`: 페이지 하단 영역 (저작권, 링크 등)

> **쉽게 말하면:** 집을 지을 때 벽을 세우기 전에 기둥부터 세우잖아요. `<header>`, `<main>`, `<footer>`는 웹 페이지의 기둥 역할이에요. 이 뼈대를 먼저 잡아두면 나중에 내용을 채우기가 훨씬 쉬워요!

## 3-7. favicon(파비콘)이 설정되어 있지 않아요 (중요도: 낮음)

### 문제

`index.html`의 `<head>`에 favicon 링크가 없어요.

### 해결 방법

**Step 1.** 프로젝트 로고나 간단한 아이콘 파일을 준비하세요. SVG 형식이 가장 좋아요 (크기 조절이 자유롭고 용량이 작아요).

**Step 2.** 아이콘 파일을 프로젝트 루트(`/`)에 놓고, `index.html`의 `<head>` 안에 링크를 추가하세요.

```html
<!-- 수정 전 (index.html 3~7번째 줄) -->
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>latte-bnb</title>
</head>

<!-- 수정 후 -->
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <link rel="icon" type="image/svg+xml" href="/favicon.svg" />
  <title>라떼비앤비 - 특별한 숙소를 찾아보세요</title>
</head>
```

> **쉽게 말하면:** 파비콘은 브라우저 탭에 보이는 작은 아이콘이에요. 앱 아이콘 같은 거죠. 없으면 밋밋한 기본 아이콘이 보이니까, 우리만의 아이콘을 넣어주면 훨씬 프로페셔널해 보여요!
