## 이슈 1: API 호출을 공통 함수로 통일하기

### 문제

프로젝트에는 이미 `src/api/client.js`에 `request()`와 `authRequest()` 함수가 준비되어 있습니다. 이 함수들은 토큰 처리, 헤더 설정, 에러 처리를 자동으로 해줍니다. 그런데 여러 페이지에서 이 함수를 사용하지 않고 `fetch()`를 직접 호출하고 있습니다. 같은 로직이 여러 곳에 흩어져 있으면 나중에 API 주소가 바뀌거나 헤더 형식이 달라질 때 모든 파일을 하나하나 찾아서 수정해야 합니다.

`src/api/client.js`의 두 함수 역할을 먼저 이해하겠습니다.

- `request(endpoint, options)` -- 토큰이 있으면 자동으로 Authorization 헤더를 붙여줍니다. 로그인 여부와 상관없이 호출할 수 있습니다.
- `authRequest(endpoint, options)` -- 토큰이 없으면 `null`을 반환하고 요청을 보내지 않습니다. 로그인이 필수인 API에 사용합니다.

### 수정 파일 1: reservations-check/index.js

현재 `listApi` 함수가 토큰을 매개변수로 받아서 직접 `fetch`를 호출합니다.

```js
// 수정 전
import constants from '../src/constants.js';

const API_BASE = constants.API_BASE_URL;

async function listApi(token) {
  const res = await fetch(`${API_BASE}/me/reservations`, {
    method: 'GET',
    headers: {
      'Content-Type': 'application/json',
      Authorization: `Bearer ${token}`,
    },
  });

  if (!res.ok) {
    throw new Error(`HTTP 오류: ${res.status}`);
  }

  return res.json();
}
```

`authRequest`를 사용하면 토큰 처리, 헤더 설정, 에러 처리가 모두 자동으로 됩니다. `listApi` 함수가 한 줄로 줄어듭니다.

```js
// 수정 후
import { authRequest } from '../src/api/client.js';

async function listApi() {
  return authRequest('/me/reservations');
}
```

`loadReservation` 함수에서 `listApi(token)` 호출 부분도 함께 수정합니다.

```js

const token = getToken();

  showError(`로그인이 필요합니다.`);
  return;
}

```

```js
// 수정 후
const data = await listApi();
if (!data) {
  showError('로그인이 필요합니다.');

```

## 이슈 2: 중복된 인증 코드를 공통 모듈로 통합하기

### 문제

`reservations-detail/index.js`에서 `getToken()` 함수를 자체적으로 정의하고 있습니다. 그런데 `src/utils/auth.js`에 이미 동일한 기능의 `getToken()`이 있습니다. 같은 기능을 두 곳에서 따로 만들면, 나중에 토큰 저장 방식이 바뀔 때(예를 들어 localStorage 대신 sessionStorage를 쓰기로 할 때) 하나만 수정하고 나머지를 빠뜨리는 실수가 생깁니다.

### 해결 방법

**Step 1.** 자체 정의한 `getToken()`과 `storageKeys` 객체를 삭제하고, 공통 모듈에서 import합니다.

```js
// 수정 전 (reservations-detail/index.js 상단)
import constants from '../src/constants.js';
const API_BASE = constants.API_BASE_URL;

const storageKeys = {
  token: 'accessToken',
  reservationId: 'reservationId',
};

function getToken() {
  return localStorage.getItem(storageKeys.token);
}
```

```js
// 수정 후
import { getToken } from '../src/utils/auth.js';
```

**Step 2.** `reservationId`를 localStorage 대신 URL 파라미터로 전달하도록 개선합니다.

현재는 예약 목록 페이지에서 예약을 클릭하면 `localStorage.setItem('reservationId', id)`로 저장하고, 상세 페이지에서 `localStorage.getItem('reservationId')`로 꺼내 씁니다. 이 방식의 문제점은 사용자가 브라우저 탭을 여러 개 열면 마지막에 클릭한 예약만 보인다는 것입니다. URL 파라미터 방식은 각 탭이 독립적으로 동작합니다.

예약 목록 페이지(`reservations-check/index.js`)에서 클릭 이벤트를 수정합니다.

```js
// 수정 전 (reservations-check/index.js)
item.addEventListener('click', () => {
  localStorage.setItem('reservationId', id);
  location.href = `../reservations-detail/`;
});
```

```js
// 수정 후
item.addEventListener('click', () => {
  location.href = `../reservations-detail/?id=${id}`;
});
```

상세 페이지(`reservations-detail/index.js`)에서 URL 파라미터로 받도록 수정합니다.

```js
// 수정 전
function getReservationId() {
  return localStorage.getItem(storageKeys.reservationId);
}

function clearReservationId() {
  localStorage.removeItem(storageKeys.reservationId);
}
```

```js
// 수정 후
const params = new URLSearchParams(location.search);
const reservationId = params.get('id');
```

`getReservationId()` 호출부를 `reservationId` 변수로, `clearReservationId()` 호출은 삭제하면 됩니다. URL 파라미터는 삭제할 필요가 없기 때문입니다.

## 이슈 4: 비밀번호 토글을 클릭 방식으로 변경하기

### 문제

현재 비밀번호 보기 버튼이 `mouseover`(마우스를 올릴 때)와 `mouseout`(마우스를 뺄 때) 이벤트로 동작합니다. 이 방식에는 두 가지 문제가 있습니다.

첫째, 모바일 기기에는 마우스가 없으므로 `mouseover` 이벤트가 발생하지 않습니다. 스마트폰에서 비밀번호를 확인할 방법이 없습니다.

둘째, 마우스를 올리는 동안만 비밀번호가 보이므로, 긴 비밀번호를 확인하기 어렵습니다.

### 해결 방법

`click` 이벤트로 변경하면 데스크톱과 모바일 모두에서 동작합니다. 한 번 클릭하면 비밀번호가 보이고, 다시 클릭하면 숨겨지는 토글 방식입니다.

**login/index.js와 signup/index.js 두 파일 모두 수정합니다.**

```js
// 수정 전
togglePass.addEventListener('mouseover', () => {
  formElements.pw.type = 'text';
});
togglePass.addEventListener('mouseout', () => {
  formElements.pw.type = 'password';
});
```

```js
// 수정 후
togglePass.addEventListener('click', () => {
  const isPassword = formElements.pw.type === 'password';
  formElements.pw.type = isPassword ? 'text' : 'password';
});
```

- `formElements.pw.type`이 현재 `'password'`이면 `'text'`로 바꾸고, `'text'`이면 다시 `'password'`로 바꿉니다.
- `isPassword ? 'text' : 'password'`는 삼항 연산자입니다. `isPassword`가 `true`이면 `'text'`를, `false`이면 `'password'`를 반환합니다.

## 이슈 5: console.log 제거하기

### 문제

개발 중에 디버깅 용도로 넣은 `console.log`가 남아 있습니다. 특히 `signup/index.js` 152줄의 `console.log(signupData)`는 사용자가 입력한 비밀번호가 브라우저 개발자 도구 콘솔에 그대로 노출됩니다. 누군가 화면을 함께 보고 있거나, 공용 컴퓨터에서 사용하는 경우 비밀번호가 유출될 수 있습니다.

### 해결 방법

해당 줄을 삭제합니다.

**signup/index.js (152줄)**

```js
// 수정 전
const signupData = getsignupData();
console.log(signupData);  // 이 줄을 삭제합니다
const validation = validationData(signupData);
```

```js
// 수정 후
const signupData = getsignupData();
const validation = validationData(signupData);
```

**wishlist/index.js (74~76줄)**

```js
// 수정 전
if (!success) {
  console.log('찜 일괄 체크 실패');
  console.log(message);
  return;
}
```

이 부분은 사용자에게 보여줄 필요가 없는 디버깅 로그입니다. 삭제하거나, 에러 상황이므로 조용히 무시하면 됩니다.

```js
// 수정 후
if (!success) {
  return;
}
```

## 이슈 6: 인라인 스타일을 TailwindCSS 클래스로 교체하기

### 문제

프로젝트는 TailwindCSS를 사용하고 있지만, 일부 코드에서 `element.style.color = 'red'`이나 `element.style.display = 'flex'` 같은 인라인 스타일을 직접 조작합니다. TailwindCSS를 사용하면서 인라인 스타일을 섞어 쓰면, 나중에 디자인을 수정할 때 CSS 클래스와 인라인 스타일 중 어디를 고쳐야 하는지 혼란스러워집니다. 인라인 스타일은 CSS 클래스보다 우선순위가 높아서, 클래스를 변경해도 적용되지 않는 문제도 발생합니다.

### 해결 방법

**reservations-check/index.js의 에러 표시**

```js
// 수정 전
function showError(message) {
  elements.result.textContent = message;
  elements.result.style.color = 'red';
}

function clearMessage() {
  elements.result.textContent = '';
  elements.result.style.color = '';
}
```

```js
// 수정 후
function showError(message) {
  elements.result.textContent = message;
  elements.result.classList.add('text-negative-500');
}

function clearMessage() {
  elements.result.textContent = '';
  elements.result.classList.remove('text-negative-500');
}
```

- `text-negative-500`은 이 프로젝트에서 이미 사용 중인 빨간색 텍스트 클래스입니다.
- `classList.add()`는 기존 클래스를 유지하면서 새 클래스를 추가합니다.
- `classList.remove()`는 해당 클래스만 제거하고 나머지는 그대로 둡니다.

**reservation-request/reservation-request.js의 모달 토글**

모달 열기/닫기에서 `style.display`를 직접 변경하는 코드가 6곳 있습니다. TailwindCSS의 `hidden`과 `flex` 클래스를 토글하는 방식으로 변경합니다.

먼저 HTML에서 모달 요소에 기본적으로 `hidden` 클래스가 있는지 확인합니다. 없다면 추가합니다.

```js
// 수정 전 -- 모달 열기
calModal.style.display = 'flex';
document.body.style.overflow = 'hidden';
```

```js
// 수정 후 -- 모달 열기
calModal.classList.remove('hidden');
calModal.classList.add('flex');
document.body.classList.add('overflow-hidden');
```

```js
// 수정 전 -- 모달 닫기
calModal.style.display = '';
document.body.style.overflow = '';
```

```js
// 수정 후 -- 모달 닫기
calModal.classList.add('hidden');
calModal.classList.remove('flex');
document.body.classList.remove('overflow-hidden');
```

`guestModal`도 동일한 방식으로 수정합니다. 파일 내에서 `style.display`와 `style.overflow`를 검색하면 수정할 위치를 모두 찾을 수 있습니다.

## 이슈 8: innerHTML 대신 안전한 DOM 조작 사용하기

### 문제

`reservations-check/index.js`의 `createReservationItem` 함수에서 서버로부터 받은 데이터를 `innerHTML`로 직접 삽입합니다. 서버 응답에 악의적인 HTML이 포함되어 있으면 XSS(Cross-Site Scripting) 공격이 가능합니다. 예를 들어 숙소 제목에 `<script>alert('해킹')</script>`이 들어 있으면, `innerHTML`은 이것을 HTML로 해석하여 스크립트를 실행합니다.

`reservations-detail/index.js`의 `createDetail` 함수도 동일한 문제가 있습니다.

### 해결 방법

`textContent`로 텍스트를 삽입하면 HTML 태그가 해석되지 않아 안전합니다. `src`나 `alt` 같은 속성은 직접 속성으로 설정합니다.

**reservations-check/index.js**

```js
// 수정 전
function createReservationItem(reservation) {
  const { accommodation, schedule, id } = reservation;
  const checkIn = formatMonthDay(schedule.checkInDate);
  const checkOut = formatMonthDay(schedule.checkOutDate);

  const item = document.createElement('li');
  item.className = 'w-full max-w-[600px] mx-auto';

  item.innerHTML = `
    <img class="w-full h-40 object-cover rounded-md"
    src="${accommodation.thumbnailUrl}"
    alt="${accommodation.title}">
    <p class="mt-2 font-bold">${accommodation.title}</p>
    <p class="text-sm text-gray-500">${checkIn} ~ ${checkOut} | ${schedule.nights}박</p>
  `;

  item.addEventListener('click', () => {
    localStorage.setItem('reservationId', id);
    location.href = `../reservations-detail/`;
  });

  return item;
}
```

```js
// 수정 후
function createReservationItem(reservation) {
  const { accommodation, schedule, id } = reservation;
  const checkIn = formatMonthDay(schedule.checkInDate);
  const checkOut = formatMonthDay(schedule.checkOutDate);

  const item = document.createElement('li');
  item.className = 'w-full max-w-[600px] mx-auto';

  const img = document.createElement('img');
  img.className = 'w-full h-40 object-cover rounded-md';
  img.src = accommodation.thumbnailUrl;
  img.alt = accommodation.title;

  const titleP = document.createElement('p');
  titleP.className = 'mt-2 font-bold';
  titleP.textContent = accommodation.title;

  const scheduleP = document.createElement('p');
  scheduleP.className = 'text-sm text-gray-500';
  scheduleP.textContent = `${checkIn} ~ ${checkOut} | ${schedule.nights}박`;

  item.appendChild(img);
  item.appendChild(titleP);
  item.appendChild(scheduleP);

  item.addEventListener('click', () => {
    location.href = `../reservations-detail/?id=${id}`;
  });

  return item;
}
```

- `document.createElement()`로 요소를 만들고, `textContent`로 텍스트를 넣으면 HTML 태그가 자동으로 이스케이프됩니다.
- `img.src = url`처럼 속성을 직접 설정하는 것은 안전합니다.

만약 복잡한 구조에서 `innerHTML`을 사용해야 하는 상황이라면, 최소한 이스케이프 함수를 만들어서 사용합니다.

```js
// 이스케이프 유틸리티 함수
function escapeHtml(str) {
  const div = document.createElement('div');
  div.textContent = str;
  return div.innerHTML;
}

// 사용 예
item.innerHTML = `<p>${escapeHtml(accommodation.title)}</p>`;
```

이 함수는 `textContent`에 문자열을 넣으면 HTML 특수문자(`<`, `>`, `&` 등)가 자동으로 이스케이프되는 원리를 이용합니다. 그 결과를 `innerHTML`로 읽으면 안전한 문자열을 얻을 수 있습니다.

## 이슈 9: Enter 키 처리 간소화

### 문제

`login/index.js`와 `signup/index.js`에 `enterPress()` 함수가 있습니다. 이 함수는 각 입력 필드에서 Enter를 누르면 다음 필드로 포커스를 이동하고, 마지막 필드에서는 제출 버튼을 클릭하는 역할을 합니다.

`<form>` 태그 안의 `<input>`에서 Enter를 누르면 브라우저가 자동으로 `submit` 이벤트를 발생시킵니다. 마지막 필드에서 버튼의 `click`을 호출하는 부분은 이 기본 동작과 중복됩니다.

### 해결 방법

입력 필드 사이를 Enter로 이동하는 기능을 유지하고 싶다면, 현재 코드를 그대로 두되 마지막 필드에서 `formElements.loginbtn.click()` 대신 `form.requestSubmit()`을 사용하는 것이 더 적절합니다.

```js
// 수정 전
if (nextForm) {
  nextForm.focus();
} else {
  formElements.loginbtn.click();
}
```

```js
// 수정 후
if (nextForm) {
  nextForm.focus();
} else {
  loginForm.requestSubmit();
}
```

`requestSubmit()`은 `submit()` 이벤트 핸들러를 거치면서 폼을 제출합니다. 버튼의 `click()`을 호출하는 것보다 의미가 명확합니다.

Enter로 필드를 이동하는 기능이 필요 없다면, `enterPress()` 함수 전체를 삭제해도 됩니다. 브라우저 기본 동작으로 마지막 필드에서 Enter를 누르면 폼이 제출됩니다.

## 이슈 10: wishlist에서 공통 API 함수 사용하기

### 문제

`wishlist/index.js`의 `checkWished()` 함수가 `fetch`를 직접 호출하여 찜 목록을 확인합니다. 그런데 `src/api/auth.js`에 이미 동일한 기능의 `checkWish()` 함수가 있고, `landing.js`에서는 이 함수를 사용하고 있습니다. 같은 프로젝트 안에서 같은 API를 호출하는 방식이 파일마다 다르면 유지보수가 어렵습니다.

### 해결 방법

```js
// 수정 전 (wishlist/index.js)
async function checkWished() {
  const accommodationIds = [...roomData.keys()];
  const res = await fetch(`${constants.API_BASE_URL}/me/wishlist/check`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      Authorization: `Bearer ${token}`,
    },
    body: JSON.stringify({ accommodationIds: accommodationIds }),
  });

  const { success, message, data } = await res.json();

  if (!success) {
    console.log('찜 일괄 체크 실패');
    console.log(message);
    return;
  }

  for (let id of data.wishlistedAccommodationIds) {
    roomData.get(id).setWish(true);
  }
}
```

```js
// 수정 후
import { checkWish } from '../src/api/auth.js';

async function checkWished() {
  const res = await checkWish([...roomData.keys()]);
  if (!res) return;

  for (let id of res.data.wishlistedAccommodationIds) {
    roomData.get(id).setWish(true);
  }
}
```

- `checkWish()` 함수는 내부에서 `authRequest`를 사용하므로, 토큰 처리와 에러 처리가 자동으로 됩니다.
- `console.log`도 함께 제거되어 이슈 5의 수정 사항도 반영됩니다.
- `landing.js`와 동일한 패턴을 사용하므로, 프로젝트 전체에서 일관성이 유지됩니다.

## 이슈 11: 모달 토글 방식 통일하기

### 문제

프로젝트 안에서 모달을 열고 닫는 방식이 두 가지로 나뉘어 있습니다.

1. `reservation-request/reservation-request.js`는 `style.display`를 직접 변경합니다.
2. `profile/profile.js`는 `classList`로 CSS 클래스를 토글합니다.

같은 프로젝트에서 같은 동작을 다른 방식으로 구현하면, 새로운 모달을 추가할 때 어떤 방식을 따라야 하는지 혼란스럽습니다.

### 해결 방법

`classList` 방식으로 통일합니다. TailwindCSS를 사용하는 프로젝트에서는 클래스 기반이 자연스럽고, HTML에서 초기 상태를 확인할 수 있어서 디버깅이 편합니다.

HTML에서 모달의 기본 상태를 `hidden`으로 설정합니다.

```html
<!-- 모달 기본 상태 -->
<div id="modal-calendar" class="hidden fixed inset-0 items-center justify-center bg-black/50 z-50">
  <!-- 모달 내용 -->
</div>
```

JS에서는 `hidden`과 `flex` 클래스를 토글합니다.

```js
// 모달 열기
function openModal(modal) {
  modal.classList.remove('hidden');
  modal.classList.add('flex');
  document.body.classList.add('overflow-hidden');
}

// 모달 닫기
function closeModal(modal) {
  modal.classList.add('hidden');
  modal.classList.remove('flex');
  document.body.classList.remove('overflow-hidden');
}
```

이 함수를 만들어두면 모달이 여러 개 있어도 동일한 방식으로 열고 닫을 수 있습니다.

```js
// 사용 예
document.getElementById('btn-date').addEventListener('click', () => {
  openModal(calModal);
});

document.getElementById('modal-calendar-cancel').addEventListener('click', () => {
  closeModal(calModal);
});
```

## 이슈 12: emptyState 컴포넌트 개선

### 문제

`src/components/emptyState.js`의 `buildEmptyState` 함수에서 `innerHTML`을 사용하고 있습니다. 이 경우 서버 데이터가 직접 들어가지 않아 XSS 위험은 낮지만, `innerHTML`로 구조를 만든 뒤 `querySelector`로 다시 찾아서 속성을 설정하는 방식은 비효율적입니다. 처음부터 DOM API로 만들면 코드가 더 명확해집니다.

### 해결 방법

```js
// 수정 전
export function buildEmptyState(message) {
  const div = document.createElement('div');
  div.className =
    'w-full min-h-140 col-span-3 flex gap-20 items-center justify-center text-lg font-semibold text-shark-600';
  div.innerHTML = `${message} <span></span> 비어있습니다.`;

  const highlight = div.querySelector('span');
  highlight.className =
    'inline-block text-[10rem] font-extrabold text-primary-500 align-text-top';
  highlight.textContent = '텅...';

  return div;
}
```

```js
// 수정 후
export function buildEmptyState(message) {
  const div = document.createElement('div');
  div.className =
    'w-full min-h-140 col-span-3 flex gap-20 items-center justify-center text-lg font-semibold text-shark-600';

  const text1 = document.createTextNode(`${message} `);

  const highlight = document.createElement('span');
  highlight.className =
    'inline-block text-[10rem] font-extrabold text-primary-500 align-text-top';
  highlight.textContent = '텅...';

  const text2 = document.createTextNode(' 비어있습니다.');

  div.appendChild(text1);
  div.appendChild(highlight);
  div.appendChild(text2);

  return div;
}
```

- `document.createTextNode()`로 텍스트 노드를 만들면, `innerHTML` 없이 텍스트와 요소를 조합할 수 있습니다.
- `querySelector`로 다시 찾는 과정이 없어져 코드 흐름이 위에서 아래로 자연스럽게 읽힙니다.
